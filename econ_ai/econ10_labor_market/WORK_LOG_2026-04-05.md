## Main Dataset

- Source: U.S. Bureau of Labor Statistics (BLS)
- Program: Local Area Unemployment Statistics (LAUS)
- File used: `la.data.3.AllStatesS`
- Local raw file: `data/la.data.3.AllStatesS.txt`
- Key rule confirmed today: only `series_id` ending in `003` should be used for the unemployment-rate analysis in this project.

## What We Did Today

### 1. Set up the project structure

- Created the `econ10_labor_market/` project folder.
- Saved the raw BLS file under `data/`.
- Used `output/` for generated figures.
- Kept the Quarto report, Python exploration script, and rendered outputs together in one folder.

### 2. Used AI to design the workflow

AI was used to help:

- identify a suitable macro/labor dataset for ECON10,
- propose a folder structure,
- draft Python cleaning and plotting code,
- explain what the BLS series IDs mean,
- diagnose suspicious charts,
- revise the visualization logic,
- and organize the final report in Quarto.

Important point from today: AI helped with coding and debugging, but the final decisions came from checking the data definitions and the generated outputs carefully.

### 3. Built the exploratory Python workflow

In `explore.py`, today we:

- loaded the BLS text file with `pandas`,
- stripped extra whitespace from columns and `series_id`,
- filtered to unemployment-rate series only using the `003` suffix rule,
- extracted state FIPS codes from `series_id`,
- mapped FIPS codes to state names and abbreviations,
- parsed `year` and `period` into a monthly date column,
- converted the unemployment-rate field to numeric safely,
- and created cleaned state-level monthly time-series data.

### 4. Investigated a suspicious national trend

One of the most important debugging moments today was a chart that looked wrong.

What happened:

- an early national trend plot showed an unrealistic pattern,
- we treated that figure as a warning sign rather than accepting it,
- AI suggested a possible fix,
- but we did not stop there,
- we checked the BLS documentation and the series definitions directly.

What we concluded:

- mixing multiple LAUS measure codes can create misleading plots,
- the unemployment-rate analysis in this project must use `series_id` ending in `003` only,
- and recent months also need a coverage check because incomplete state coverage can create artificial spikes.

This was the strongest example today of how AI can help, but verification still matters.

### 5. Improved the national average logic

To avoid false spikes in the national average, we:

- computed the monthly average across states,
- counted how many states were available in each month,
- kept only months with broad coverage,
- and regenerated the national trend chart.

This produced a more credible national series for teaching and reporting.

## Main Lessons From Today

- A chart that looks polished can still be wrong.
- If something looks strange, stop and check the data definition.
- AI can speed up coding, but it cannot replace responsibility.
